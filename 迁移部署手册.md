你队友**不需要先自己再装一遍“全新的 advanced 模板”再复制代码**。最标准、最省事、最不容易出错的方式是：

> ✅ **直接 `git clone` 你们的仓库 → `composer install` → 初始化 env → 配置数据库 → `php yii migrate` → 配置虚拟主机/端口 → 访问前后台。**

下面我给你一份“队友照着做就能跑”的**本地部署流程（XAMPP/Windows 版）**，你把它原样写进仓库 README 里就行。

------

# 0. 队友本地需要准备的环境

- PHP（建议 7.4/8.0/8.1，与你项目一致）
- Composer（必须）
- MySQL（XAMPP 自带即可）
- Git（必须）
- Apache（XAMPP）或 Nginx（可选）

------

# 1. 从 GitHub 拉取项目（最重要）

在 `htdocs` 里执行（例子）：

```
cd D:\xampp\htdocs
git clone <你的仓库地址> advanced
cd advanced
```

> 注意：仓库里应该已经包含你们整个 advanced 项目目录结构（common/backend/frontend/console/vendor 不在仓库）。

------

# 2. 安装依赖（composer）

在项目根目录执行：

```
composer install
```

如果很慢可以用国内镜像（可写进 README）：

```
composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/
composer install
```

------

# 3. 初始化环境文件（advanced 必做）

在项目根目录：

```
php init
```

然后它会问：

- 选择 Development（开发环境）=> 输入 `0`（通常是 0）
- 确认覆盖 => 输入 `yes`

这一步会生成：

- `common/config/main-local.php`
- `backend/config/main-local.php`
- `frontend/config/main-local.php`
- `console/config/main-local.php`
	 等本地配置文件（一般不提交到 git）。

------

# 4. 配置数据库连接（main-local.php）

队友需要在 `common/config/main-local.php` 配置自己的数据库账号密码。

找到：

```
'db' => [
    'class' => 'yii\db\Connection',
    'dsn' => 'mysql:host=localhost;dbname=yii2advanced',
    'username' => 'root',
    'password' => '',
    'charset' => 'utf8',
],
```

------

# 6. 执行 migration（把表结构全部建出来）

在项目根目录执行：

```
php yii migrate
```

看到所有 migration applied 就 OK。

> 这一步是你作为组长最该保证的：**所有表都能靠 migrate 自动生成**，队友不用手动建表。

------

# 7. 生成 key（如果你们启用了 cookieValidationKey）

通常 advanced 初始化时会生成，但有时候会缺。检查：

- `common/config/main-local.php`
- `frontend/config/main-local.php`
- `backend/config/main-local.php`

里面有：

```
'cookieValidationKey' => '...'
```

如果为空会报错，需要填随机字符串。
