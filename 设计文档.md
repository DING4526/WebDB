# 《烽火记忆》

## 抗战胜利80周年 · 时间轴与人物数据库系统

**项目规划与需求说明（完整版）**

------

## 一、项目背景与意义

2025 年是中国人民抗日战争胜利 80 周年。为铭记历史、缅怀先烈，本项目以“史实时间轴 + 人物群像”为核心，通过数据库系统对抗战重要事件与人物进行系统化整理与展示，实现抗战历史的数字化表达。

本项目同时作为《互联网数据库开发》课程设计，重点体现：

- 数据库建模能力
- 前后台系统设计能力
- MVC 架构实践
- 数据可视化与交互展示能力

------

## 二、项目总体目标

构建一个 **基于 Web 的抗战历史展示与管理平台**，实现：

- 前台：
	- 抗战史实时间轴展示
	- 抗战人物卡片与详情页
	- 留言纪念互动
- 后台：
	- 事件与人物信息的管理与发布
	- 数据关联维护
	- 简单统计分析与图表展示

系统结构清晰、功能完整、技术难度可控，同时具备良好的展示效果。

------

## 三、系统功能需求分析

### 3.1 前台功能需求

#### （1）首页 —— 抗战史实时间轴

- 按时间顺序展示抗战重要事件
- 支持按年份/阶段筛选（如 1931–1936、1937–1945）
- 每条事件包含：
	- 时间
	- 标题
	- 简要描述
	- 关联标签
	- 查看详情入口

#### （2）事件详情页

- 展示事件完整描述
- 显示关联人物（人物卡片形式）
- 显示事件相关图片/资料
- 支持留言或评论（可选）

#### （3）人物库页面

- 以卡片墙形式展示抗战人物
- 人物卡片信息：
	- 姓名
	- 身份类型
	- 简要介绍
- 支持点击进入人物详情页

#### （4）人物详情页

- 人物生平简介
- 参与的抗战事件列表（按时间排序）
- 纪念留言区

#### （5）纪念留言墙

- 普通用户可提交纪念留言
- 留言内容在后台审核后展示

------

### 3.2 后台功能需求

#### （1）管理员登录

- 管理员账号登录
- 基本权限控制

#### （2）事件管理模块

- 新增、编辑、删除抗战事件
- 设置事件发布时间、排序
- 事件与人物的关联维护

#### （3）人物管理模块

- 新增、编辑、删除人物信息
- 人物分类（将领 / 士兵 / 群众 / 记者等）

#### （4）留言管理模块

- 查看用户留言
- 审核 / 删除留言

#### （5）数据统计模块（加分项）

- 按年份统计事件数量（折线图/柱状图）
- 按人物身份统计数量分布（饼图/柱状图）

------

## 四、数据库设计规划（不少于 10 张表）

### 核心业务表

1. `user` —— 管理员用户表
2. `event` —— 抗战事件表
3. `person` —— 抗战人物表
4. `event_person` —— 事件-人物关联表
5. `message` —— 纪念留言表

### 辅助与扩展表

1. `tag` —— 标签表
2. `event_tag` —— 事件-标签关联
3. `person_tag` —— 人物-标签关联
4. `media` —— 图片/资料表
5. `visit_log` —— 访问统计表
6. `admin_log` —— 管理操作日志
7. `team_member` —— 团队成员展示表

> 数据库表数量充足，关系清晰，符合课程要求。

------

## 五、技术实现方案（低难度版）

- 开发框架：Yii2（MVC 架构）
- 数据库：MySQL
- 前端：HTML + CSS + 简单 JS
- 后台：Yii2 Gii 生成 CRUD
- 图表：简单 JS 图表库（如 Chart.js）

**不涉及地图、不涉及复杂算法、不涉及爬虫，确保按时完成**
